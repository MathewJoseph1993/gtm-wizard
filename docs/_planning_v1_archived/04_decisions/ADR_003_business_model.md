# ADR-003: Business Model Decision

> **Status:** Pending Research
> **Decision Date:** TBD
> **Last Updated:** December 25, 2024

## Context

GTM Wizard needs a sustainable business model that balances open source community building with revenue generation.

## Options Considered

### Option A: Pure Open Source + Sponsorship
- Everything free forever
- Revenue from GitHub Sponsors, grants
- Example: curl, SQLite

### Option B: Open Core
- Core features free
- Premium features paid
- Example: GitLab, Elastic

### Option C: Open Source + Hosted SaaS
- Self-host free
- Pay for hosted convenience
- Example: Supabase, PostHog

### Option D: Open Source + Services
- Code free
- Consulting/services paid
- Example: Many consultancies

### Option E: Dual License
- Free for non-commercial
- Paid for commercial use
- Example: HashiCorp (BSL)

## Decision Criteria

| Criterion | Weight | Notes |
|-----------|--------|-------|
| Community growth | 20% | Does model help adoption? |
| Revenue potential | 25% | Can we sustain? |
| Revenue speed | 15% | How fast to first $? |
| Complexity | 15% | Can we execute? |
| OSS credibility | 15% | Community perception |
| Visa evidence | 10% | Open source contribution value |
